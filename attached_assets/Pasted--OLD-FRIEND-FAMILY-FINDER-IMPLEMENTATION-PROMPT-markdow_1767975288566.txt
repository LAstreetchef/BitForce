# **OLD FRIEND & FAMILY FINDER - IMPLEMENTATION PROMPT**

```markdown
**TASK:** Create an "Old Friend & Family Finder" tool that helps users locate people using public data sources and compliant APIs.

**PRODUCT NAME:** Connection Finder AI Buddy

**TAGLINE:** "Reconnect with people from your past using smart search technology"

**CORE FEATURES:**

1. **INTELLIGENT SEARCH INPUT:**
   - Name (required)
   - Last known city/state (optional)
   - Age or birth year (optional)
   - Previous addresses (optional)
   - Known relatives (optional)
   - Years of association (optional)

2. **MULTI-SOURCE SEARCH ENGINE:**
   - Query 5+ data sources simultaneously:
     1. Public records databases (via Whitepages Pro API)
     2. Social media profiles (public data only)
     3. Professional networks (LinkedIn public profiles)
     4. Property ownership records (where publicly accessible)
     5. Voter registration data (public records)
   - Return aggregated results with duplicates removed

3. **CONFIDENCE SCORING SYSTEM:**
   ```
   Scoring Algorithm:
   - Exact name match: +40 points
   - Location match (city/state): +25 points
   - Age within 3 years: +20 points
   - Relative name match: +15 points
   - Previous address match: +10 points
   
   Confidence Levels:
   90-100% = "Very High Confidence"
   70-89% = "High Confidence"
   50-69% = "Moderate Confidence"
   30-49% = "Low Confidence"
   0-29% = "Possible Match"
   ```

4. **RESULTS DISPLAY:**
   - Clear match cards for each potential person
   - Confidence percentage prominently displayed
   - Available information shown (location, age, relatives)
   - Source indicators (public record, social media, etc.)
   - "Save Result" and "Export Report" options

**TECHNICAL IMPLEMENTATION:**

**FRONTEND (React/HTML/CSS):**
1. **Search Interface:**
   - Clean, multi-field form with clear labels
   - Progressive disclosure (basic → advanced fields)
   - Real-time form validation
   - Consent checkbox required: "I agree to use this service for permissible purposes including locating friends, family, or colleagues I have a prior relationship with"

2. **Results Dashboard:**
   - Grid/List view toggle
   - Filter by confidence score
   - Sort by match relevance
   - Pagination for large result sets

3. **User Dashboard:**
   - Saved searches section
   - Search history (30-day retention)
   - Export options (PDF, CSV)

**BACKEND (Python/Flask):**
1. **API Architecture:**
   ```python
   # Main search endpoint
   POST /api/search-person
   # Request: {name, location, dob, relatives, etc.}
   # Response: {matches: [], search_id, confidence_scores}
   
   # Data source endpoints
   GET /api/search-whitepages
   GET /api/search-social
   GET /api/search-public-records
   ```

2. **Data Processing:**
   - Deduplication engine (merge similar results)
   - Confidence scoring calculator
   - Result caching (24 hours)
   - Rate limiting (per user/IP)

3. **External API Integrations:**
   - Whitepages Pro API (primary paid source)
   - LinkedIn Public Profile API
   - Facebook Public Page Graph API
   - Twitter Users Lookup API
   - County property records (web scraping where allowed)

**COMPLIANCE & LEGAL PROTECTIONS:**

1. **REQUIRED USER AGREEMENTS:**

   - Privacy policy acknowledgment

2. **DATA HANDLING:**

   - Regular data purging schedule

3. **DISCLAIMERS & TRANSPARENCY:**
   - Clear labeling: "Public Records Search Tool"
   - Disclaimer: "Results may contain inaccuracies. Verify through multiple sources."
   - Source attribution for all data
   - "Report Inaccuracy" button on each result
